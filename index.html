1. Principle of Local-First Operation

Project Z operates strictly as a local-first application client framework. All operational workflows, data processing pipelines, log parsing scripts, stack trace maps, and telemetry ingest vectors execute entirely on your physical iOS or iPadOS device hardware environment using local Neural Engine frameworks.

2. Zero Data Collection & Apple Intelligence Constraints

Because Project Z functions completely independent of centralized cloud instances, we maintain an architectural framework that leaves data entirely in your custody. This application relies exclusively on hardware configured with Apple Intelligence to run advanced parsing logic locally. No raw payload configurations or calculated metrics are broadcast over network sockets; everything is computed securely on your own device silicon.

When an operator explicitly chooses to opt into our Beta Intake program by submitting their email address on this page, that data package is securely transmitted over TLS-encrypted network channels to our trusted data relay service partner, Formspree (Formspree, Inc.). Formspree processes and temporarily caches these text strings purely as a technical delivery middleman to route the notification straight to our command inbox. This intake data is never shared, never sold, and is completely decoupled from your offline client runtime computing environment.

4. Ephemeral Memory Persistence

Your processing structures live in temporary volatile client application memory maps. Data configuration logs can be completely cleared by resetting your system container or running the application flush utility. We do not use persistent network caches or silent secondary storage tracking protocols.

5. Third-Party Transfers & Regulatory Standards

Since data is never received by us, we never execute transfers, monetization procedures, or disclosure protocols to third parties. Our architecture aligns inherently with international data boundaries (including GDPR and CCPA mandates) by leaving data entirely inside the user's sovereign hardware containment perimeter.
